Mike Hartington | @mhartington

What is Ionic?

  • Cross-platform UI for building native, desktop, and web apps
  • Powers ~20% of apps in Apple and Google Play Stores
  • Used by more than 5 million developers in 200 countries

A long time ago...

  • Ionic was only Angular
  • Wanted to support other tools
  • But not rewrite over and over

Enter Web Components

~6 months of work

  • Rewrote as Web Components
    • Components are self-contained
  • Framework adaptors when needed (Routing)

Enter Ionic 4.0

Ionic For everyone

What did the people say?

  • Huge success with devs and large teams
  • Refocused our energy in different areas
  • Following framework ecosystem better

So...now what?

Challenges

  • Props/Attrs and values
  • Handling Custom Events
  • Integration with Routers

Embrace React

  • Keeping opinions minimal
  • Splitting router from components
  • Automated Wrapper Components

What about...React Native?

  • Ionic/Capacitor is web first
  • No major context shift
  • Web enhanced with Native
  • "Learn once, write anywhere", but for realz

But experiment!

  • React Native as a native wrapper
  • Embed Capacitor in RN app
  • Room for both to exist

So, what's the takeaway?

Ionic React is great

  • Best of Ionic, best of React
  • Web PLUS Native Mobile
  • Reuse Existing React/Web skills
